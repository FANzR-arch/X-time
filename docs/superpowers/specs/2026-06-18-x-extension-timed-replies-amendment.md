# X 定时回复实现修正

日期：2026-06-18

## 修正原因

能力核对确认：X 网页原生定时功能适用于原创帖子，但回复编辑器没有原生排期入口。原设计中“调用 X 原生回复排期”的前提不成立。

## 最终方案

- 原创帖继续使用 X 原生定时列表。
- 回复任务由扩展保存到 `chrome.storage.local`。
- 每条回复保存绝对时间戳；目标时区仅负责把用户输入时间换算为该时间戳。
- `chrome.alarms` 在目标时间唤醒后台 service worker。
- 后台复用已打开的 X 标签页，或创建新标签页并打开目标帖子。
- content script 只在任务到点后填写回复，并且只点击明确标记为 `Reply / 回复 / 回覆` 的按钮。
- 浏览器关闭期间错过的任务在下次启动后尽快恢复。
- 如果浏览器、扩展或标签页在发送过程中中断，任务标记为“结果未知”，不得自动重发。

本文件覆盖同日旧设计中关于“X 原生回复排期”“进入 X 定时列表”和“缺少排期按钮则停止”的部分。其余独立入口、批量格式、目标时区、双时区预览、纯文字限制及失败恢复规则保持有效。
